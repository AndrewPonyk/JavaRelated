1. What is the difference between Django's `select_related()` and `prefetch_related()`?

Answer: select_related() uses SQL JOIN for foreign key relationships, loads related objects in single query. prefetch_related() uses separate queries for many-to-many and reverse foreign key relationships, then joins in Python. Use select_related for ForeignKey/OneToOne, prefetch_related for ManyToMany/reverse FK.

```python
            initializer='zeros',
            trainable=True
        )

    def call(self, inputs):
        return tf.matmul(inputs, self.w) + self.b
```

Answer: This is a complete custom dense layer implementation. No completion needed - it properly implements build() for weight creation and call() for forward pass computation.
