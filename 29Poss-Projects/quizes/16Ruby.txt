5. Find the N+1 query problem in this Rails code:
```ruby

  def index
    @posts = Post.all
  end
end

# In the view:
<% @posts.each do |post| %>
  <h3><%= post.title %></h3>
  <p>By: <%= post.user.name %></p> <!-- N+1 problem here -->
  <p><%= post.content %></p>
<% end %>
```

Answer: Each post.user.name triggers a separate database query. Fix with eager loading: `@posts = Post.includes(:user)` or `@posts = Post.joins(:user).select('posts.*, users.name as user_name')` to load users in single query.
