3. What is the difference between Ethereum and Hyperledger Fabric?

Answer: Ethereum is public blockchain, permissionless, PoW/PoS consensus, smart contracts (Solidity), cryptocurrency (ETH), global state. Hyperledger Fabric is private/permissioned, modular architecture, pluggable consensus, chaincode (Go/Java/Node.js), no cryptocurrency, channels for privacy. Choose Ethereum for public dApps, Fabric for enterprise.

5. Find the security vulnerability in this smart contract:
```solidity
pragma solidity ^0.8.0;

contract Auction {
    address public highestBidder;
    uint256 public highestBid;

    function bid() public payable {
        require(msg.value > highestBid, "Bid too low");

        // Vulnerability: reentrancy attack possible
        payable(highestBidder).transfer(highestBid);

        highestBidder = msg.sender;
        highestBid = msg.value;
    }
}
```

Answer: Reentrancy vulnerability - external call before state update. Fix by updating state first (Checks-Effects-Interactions pattern) or use ReentrancyGuard from OpenZeppelin. Attacker could recursively call bid() to drain contract funds.

9. What is the difference between SHA-256 and Keccak-256?

Answer: SHA-256 is NIST standard, used in Bitcoin, 256-bit output, standardized algorithm. Keccak-256 is Ethereum's hash function, SHA-3 finalist (pre-standardization version), 256-bit output, used for addresses and state. Both are cryptographically secure but incompatible.

15. What is the difference between Layer 1 and Layer 2 blockchain solutions?

Answer: Layer 1 is base blockchain (Ethereum, Bitcoin), on-chain execution, higher security, lower throughput, higher fees. Layer 2 is scaling solution (Optimistic Rollups, ZK-Rollups, State Channels), off-chain execution, inherits L1 security, higher throughput, lower fees. L2 settles to L1 for finality.
