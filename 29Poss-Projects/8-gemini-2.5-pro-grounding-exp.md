### 2. Multithreading

**Big Project 1: Concurrent Web Crawler**
*   **Tech Stack:** Java, Concurrency Primitives (Locks, Semaphores), Atomic Operations, Thread Pools (ExecutorService), JSoup (for HTML parsing), Kafka.
*   **Short Tech Description:** A multi-threaded application that crawls websites starting from a seed URL. A thread pool is used to manage worker threads, each responsible for fetching a webpage, parsing its content for links, and adding new links to a shared, thread-safe queue for further crawling. Race conditions are prevented using synchronization and atomic counters for stats.
*   **Business Logic:** This tool can be used to index a website for a search engine, check for broken links, or scrape data for analysis. The concurrency allows it to process thousands of pages much faster than a single-threaded approach.

**Big Project 2: Stock Market Simulation**
*   **Tech Stack:** Python, `threading` module, `multiprocessing` module, Locks, Semaphores, Producer-Consumer pattern, PyQT/Tkinter for UI.
*   **Short Tech Description:** A simulation where multiple "trader" threads concurrently try to buy and sell stocks based on fluctuating prices generated by a "market" thread. The project implements the Producer-Consumer pattern for order processing and uses locks to protect shared resources like account balances and stock portfolios, avoiding deadlocks.
*   **Business Logic:** This application simulates a real-world stock exchange, helping to visualize and understand market dynamics, race conditions, and the importance of synchronized access to shared financial data.

**Medium Project 1: Parallel File Processor**
*   **Tech Stack:** Java, Threads, Thread Pools, Producer-Consumer pattern, NIO (for file I/O).
*   **Short Tech Description:** A tool that processes large text files by breaking them down into chunks. A main thread (producer) reads chunks of the file and places them in a blocking queue. A pool of worker threads (consumers) concurrently takes chunks from the queue to perform a task, like counting words or finding a specific pattern.
*   **Business Logic:** This utility significantly speeds up the processing of large log files or datasets for data analysis or log monitoring tasks. It demonstrates how to parallelize I/O-bound and CPU-bound tasks.

**Medium Project 2: Dining Philosophers Problem Visualization**
*   **Tech Stack:** Go, Goroutines, Channels, Mutexes, `fyne` or other GUI library.
*   **Short Tech Description:** A visual implementation of the classic Dining Philosophers concurrency problem. Each philosopher is a goroutine, and the forks are shared resources protected by mutexes. The visualization shows when philosophers are thinking, hungry, or eating, and helps demonstrate concepts like deadlock and starvation.
*   **Business Logic:** This is an educational tool used to teach and understand fundamental concurrency problems. It provides a clear visual representation of resource contention and the challenges of designing deadlock-free systems.
